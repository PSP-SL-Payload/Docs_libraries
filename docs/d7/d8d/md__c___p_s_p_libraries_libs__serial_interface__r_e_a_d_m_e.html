What is the SerialInterface abstract class?

This class is an abstract class, meaning that is kind of like a template. It isn't exactly a template, because that's a specific type of class in C++ but we're using it as a template for other SerialInterface modules. Every single function, struct, variable, etc in this class will be available in its child classes.

When should I use this class?

Making child classes

Whenever you need to create a Serial connection between a Raspberry PI and an Arduino, a child class that inherits this module can be used to interface with that connection.

For example, let's say I'm making a new child class for the Serial Interface between the GCS ButtonHandler Arduino and the GCS Core RaspberryPI.

  1. Start by creating a new child class called ButtonHandlerSerialInterface.
  2. Have it inherit this class, like so: class ButtonHandlerSerialInterface : public SerialInterface.

Any abstract function in the SerialInterface class needs to implemented in the ButtonHandlerSerialInterface class. You can find these by looking at all the fuctions of this general format in the header file virtual void foo() = 0;

Example Test Code (change names accordingly)

{C++}
#include <Arduino.h>
#include childClass
// TODO: change childClass to the respective child class, ex: ButtonHandlerSerialInterface
void setup()
{
Serial.begin(9600);
}
void loop()
{
// Whenever you are in a busy wait, the ReadSerial() method can be called to empty the
// serial input buffer one character per call.
childClass::Instance()->ReadSerial();
// To write a packet to the GCS Core Raspberry PI, the following methods can be used
// Write_TODO_Packet() where _TODO_ should be replaced with the necessary packet type.
childClass::Instance()->Write_TODO_Packet(/* Packet Parameters */);
// To write a serial string to the GCS Core Raspberry PI, the standard Arduino serial
// print methods should be used.
Serial.println("Serial String Message");
// To receive a packet from SerialPackets.
SerialPackets::serialPacketNode recv;
childClass::Instance()->WaitForPacket(recv, SerialPackets::SerialPacketIDs::_TODO_PacketID);
// To print the contents of any serial packet
childClass::Instance()->PrintPacket(recv);
}
