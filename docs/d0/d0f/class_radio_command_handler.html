◆ GeneralRadioCommands

enum RadioCommandHandler::GeneralRadioCommands : uint16_t

The supported GENERAL commands that can be transmitted via radios. They DO NOT require any additional data sent in the payload with them.

Note that these are RADIO supported, not necessarily Payload supported. That means that unless you implement this functionality on Payload, the commands will be meaningless.

These commands start at index LOWEST_INDEX_OF_GENERAL_COMMANDS to separate from ModdedRadioCommands. When adding a command, keep it in direct order, DO NOT SKIP NUMBERS as the command validity checking requires no skips. Make sure to update the index values below when adding/removing commands.

enum RadioCommandHandler::ModdedRadioCommands : uint16_t

The supported MODDED commands that can be transmitted via radios. They DO require additional data sent with them in the payload. This is usually in the form of how many steps to turn a motor, etc.

Note that these are RADIO supported, not necessarily Payload supported. That means that unless you implement this functionality on Payload, the commands will be meaningless.

These commands start at index LOWEST_INDEX_OF_MODDED_COMMANDS to separate from GeneralRadioCommands. When adding a command, keep it in direct order, DO NOT SKIP NUMBERS as the command validity checking requires no skips. Make sure to update the index values below when adding/removing commands.

RadioCommandHandler::CommandResponseStatus RadioCommandHandler::CreateAndSendGeneralCommand ( GeneralRadioCommands  radioCommand,
RadioCommandReturn &  returnedData 
)
virtual

Creates a general purpose command. This can be a simple instruction to the other side (i.e. turn LED 0 on) or it can be a data request (i.e. return altitude in meters). This method handles commands by sending them to the other side, waiting for a response if one is required, and collecting and packaging the returned data in returnedData.

Parameters
radioCommandThe Radio Command to send. This can be a data request or an instruction
returnedDataThe data returned from the command receiver. For example, if you requested IMU data, this will hold it at the return of the function
Returns
CommandResponseStatus - the response of the command. Signifies success or error and if there is available data contained in returnedData
