What is the IMU abstract class?

This class is an abstract class, meaning that is kind of like a template. It isn't exactly a template, because that's a specific type of class in C++ but we're using it as a template for other IMU modules. Every single function, struct, variable, etc in this class will be available in its child classes.

When should I use this class?

Having functions that need a IMU

Any function that needs a IMU module should take this IMU class as a parameter.

For example, let's say you have a method called bool IsPayloadStationary(IMU imu).

Notice here I have the parameter type as IMU. This means any child class of IMU can be passed in here. We could pass in an object of IMU_LSM6x, IMU_MPU6500, etc as long as the class that gets passed in is a child class of IMU.

Example:

{C++}
foo()
{
// we make an IMU_LSM6x object (a child of IMU)
IMU_LSM6x imu;
...
// we call IsPayloadStationary (notice that we pass in a IMU_LSM6x, NOT a IMU object)
IsPayloadStationary(imu);
...
}

Making child classes

Whenever you have a new IMU module, have the child class you make for it inherit this class.

For example, let's say I'm making a new child class for the MPU6500 IMU module.

  1. Start by creating a new child class called IMU_MPU6500.
  2. Have it inherit this class, like so: class IMU_MPU6500 : public IMU.

Any abstract function in the IMU class needs to implemented in the IMU_MPU6500 class. You can find these by looking at all the fuctions of this general format in the header file virtual void foo() = 0;

Now your child class can be used in the control loop while changing virtually nothing because its functionality is the same as other IMU classes.

Example Test Code (change pins and such accordingly)

{C++}
#include <Arduino.h>
#include childClass
// TODO: change childClass to the respective child class, ex: IMU_LSM6x
childClass imu;
void setup()
{
Serial.begin(9600);
imu.Init();
}
void loop()
{
DirectionalValues accel = imu.GetAccelVals();
DirectionalValues gyro = imu.GetGyroVals();
IMU::PrintReadingsAccel(accel);
IMU::PrintReadingsGyro(gyro);
delay(400);
}
