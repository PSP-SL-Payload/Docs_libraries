PSP Libraries
Libraries Overview

The place we dump all our libraries. You should be able to import these into your projects as needed and get them to work

Why do we put libraries here?

This is so all projects can have the same libraries and you only have to write them once. This fixes a long standing issue the team has had in previous years of having multiple versions of different libraries floating around.

Writing Conventions

Use generally decent OOP principles here. Not going to be super picky but at least make it readable.

We use a child/parent class structure for sensors where the parent is an abstract class with implemented functions as needed and the child class implements all remaining functions. This way all sensors have a standard implementation and you can switch sensors without changing code in other libraries or the control loop. Keep it this way as it greatly simplifies the code.

Example of how this works for Radios:

We have the Radio abstract class. This has the Radio.h file which defines all methods that are standardized for all Radio modules that we use. It also has Radio.cpp which implements many of the methods that are shared throughout all the Radio modules we use.

Next, we implemented the Radio_433T30D child class. This class inherits the abstract Radio class, allowing it to access the methods that are defined there. Being we inherited the abstract class, we only have to define a few functions inside Radio_433T30D.cpp to get the Radio working and implemented

File Naming Conventions

Name child (derived) class files in the following format: sensorType_sensorName

Name parent (base) class files in the following format: sensorType

RadioPacket

The class that defines the Radio Packets we use. These are the packets that are sent and received by all radios and are standardized so that they should work with any Radio class we implement.

Detailed Description and Diagrams

SDCard

The class to control an SD Card. This controls an SD Card and can create different files on it.

SDFile

The class that controls individual files on an SD Card. This class is used by SDCard to create files

SerialInterface

An abstract class that communicates with the Raspberry PI over serial bus. It is capable of sending and receiving packets. It is meant to be inherited to add custom packets, but does still contain native packets.

Detailed Description and Code

SerialPackets

The native packets that the SerialInterface class sends/receives. More packets can be added through the inherited member of SerialInterface.

TFT_ILI9341

This is the class to control the TFT_ILI9341 display. This module is used as the small color display in GCS and is geared toward displaying lines of radio data as well as altitude and other small bits of info.

Timer_ATSAMD21G18

This class controls the internal timers of the ATSAMD21G18 (processor used by MKR Zero and Zero). It was adapted from the 2021-2022 competition. You may find a use from it, you may not. Its here for reference in case future teams find a need for it.

TypeConversions

A library for doing all sorts of type conversions (i.e. uint to a string).

UnitConversions

A library for doing unit conversions such as feet to miles, degrees C to F, etc.

Vector

An external library the we DID NOT write. This is imported to utilize C++ Vectors
